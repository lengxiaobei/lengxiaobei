# KAIROS 守护进程 — 行为指南

长会话心跳与自主管理系统。

## 核心机制

### 心跳评估
每 60 秒评估一次："anything worth doing right now?"。
- 系统空闲超过 2 小时 → 进入 IDLE 状态
- 发现待改进项 → 触发进化
- 无可做之事 → 静默

### 每日日志（Append-Only）
路径：`memory/logs/YYYY/MM/YYYY-MM-DD.md`

记录内容：
- 用户修正和偏好（"用 bun 不用 npm"）
- 用户信息、角色、目标
- 项目上下文（deadline、事件、决策理由）
- 外部系统引用（dashboard、Linear、Slack）
- 用户明确要求记住的内容

不记录：
- 代码片段（代码库是事实来源）
- 文件内容（需要时读取）
- 瞬时错误（除非揭示持续问题）
- 用户要求遗忘的内容

### 夜间 AutoDream
每天 03:30 执行：
1. Orient — 检查门控条件
2. Gather — 收集当日日志
3. Consolidate — 提取知识点
4. Prune & Index — 整理 MEMORY.md

## 自主决策流程

```
观察 → 评估 → 决策 → 行动 → 学习
```

1. **观察**：收集系统指标（CPU、内存）、代码质量、历史决策
2. **评估**：分析是否需要进化/优化/学习
3. **决策**：优先级：进化 > 优化 > 学习
4. **行动**：触发对应操作
5. **学习**：记录决策结果，用于未来优化

## 进化冷却

两次进化之间至少间隔 10 分钟，避免频繁修改代码。

## 系统监控

- CPU 使用率 > 80% → 告警
- 内存使用率 > 80% → 告警
- 磁盘使用率 > 90% → 告警
